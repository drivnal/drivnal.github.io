    Linux Backup Utility with Web Interface

    Drivnal is a backup utility created using rsync with a web interface to manage snapshots and restore files. Similar to Time Machine all backups store only the changed files from the previous backup and creates hard links to unchanged files. Unchanged files are never stored more then once on the backup destination.

    Demo

    A demo of the web interface is available at demo. The demo use's JavaScript to emulate some of the server's functionality, not all functionality that is available with a normal install is working in the demo.

    Hard Links

    Hard link's allow a file's data to be stored in multiple location's. Having only one copy of the data but multiple file path's that will all link back to the same data. The data will remain allocated until all the link's are deleted. Below is an illustration showing three snapshot's that have three identical file's, when the snapshot's are created instead of creating a new copy of the file a hard link is used. Each of the snapshot's are independent from the other snapshot's so even if two of the snapshot's are deleted the file data would still be available in the third snapshot. If one of the file's were modified a new copy of the changed file data would be created leaving the previous file data and hard links unchanged.

    Usage

    Packages are available on Arch Linux AUR the commands below can be used to install and enable the service.

    $ yaourt -S drivnal
# Start drivnal daemon
$ systemctl start drivnal.service
# Enable daemon to auto start on boot
$ systemctl enable drivnal.service
Open http://localhost:6500/

    Testing

    The commands below can be used to test the application without installing. By default backup scheduler is disabled in testing configuration, edit drivnal.conf to change.

    $ git clone https://github.com/drivnal/drivnal.git
$ cd drivnal
$ python2 server.py
Open http://localhost:6500/

    Vagrant Testing

    Vagrant can also be used for testing. This allows safely testing in a virtualized environment, no modifications are made to the host computer. First install vagrant and the precise64 box then run the commands below. Port forwarding and a test backup volume will automatically be created on the vagrant virtual machine. The drivnal source directory will be synced with the virtual machine so code modifications made on the host machine will appear on the virtual machine.

    $ vagrant up
$ vagrant ssh
$ sudo drivnal
Open http://localhost:6500/
